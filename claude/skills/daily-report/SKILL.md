# 日報作成スキル

ユーザーの日報を対話形式で記録するスキル。**項目名や本文の書式は内部スクリプト `scripts/write_report.py` で正規化する。直接ファイルを編集してはいけない**（表記揺れや既存内容の破壊を防ぐため）。

## 保存先とフォーマット

- 保存先: `$HOME/daily_report/YYYY-MM-DD.md`（1日1ファイル、無ければ自動生成）
- 構造: 必ず以下4セクションを `##` ヘッダーで固定順に並べる

```markdown
# 日報 2026-05-11

## 今日やること

- ...

## やったこと

- ...

## 明日やること

- ...

## その他補足

- ...
```

## 記録スクリプト

スクリプトの絶対パス:

```
~/.claude/skills/daily-report/scripts/write_report.py
```

サブコマンド:

| コマンド | 用途 |
|---|---|
| `write --field <KEY> --content "<TEXT>" [--date YYYY-MM-DD] [--mode append\|overwrite]` | 指定項目に書き込み。`--mode` 既定は `append`（既存末尾に改行追記）、`overwrite` は置換 |
| `read [--date YYYY-MM-DD]` | 当該日のファイルを標準出力にそのまま表示。書き込み前の既存確認に必ず使う |
| `path [--date YYYY-MM-DD]` | 解決済みファイルパスを表示（報告用） |

`--date` 未指定時は今日（システムローカルタイム）。

## 項目キーのマッピング

ユーザーの言い回しを以下の `--field` キーに変換する。表記揺れがあっても意図に沿ってマッピングし、迷ったときだけAskUserQuestionで確認する。

| ユーザー表記の例 | `--field` |
|---|---|
| 今日やること / 今日の予定 / 今日のタスク / today | `today_plan` |
| やったこと / 完了 / 実績 / 終わったこと / done | `done` |
| 明日やること / 明日の予定 / 明日のタスク / TODO / tomorrow | `tomorrow_plan` |
| その他補足 / 補足 / メモ / 雑記 / notes | `notes` |

## 動作フロー

### 1. 対象項目を決定する

ユーザーのプロンプトに含まれる項目名から判定:

- 「日報」だけ → 4項目すべてが候補。AskUserQuestion(`multiSelect: true`) で記録したい項目を選んでもらう
- 「日報 明日やること」「日報 やったこと と 明日やること」など項目名あり → その項目のみ。確認のAskUserQuestionは不要
- 判断に迷う表現（例: 「日報 進捗」）→ AskUserQuestionで近そうな項目を提示して確認

### 2. 既存内容を確認する

書き込み前に必ず `write_report.py read --date <対象日>` を実行し、対象項目に既に内容が入っているか確認する。

### 3. 本文を収集する

対象項目ごとに、**通常のチャット（自由記述）** で本文を聞く。AskUserQuestionは選択肢が要る場面用なので、自由記述の本文収集には使わない。例:

> やったことを教えてください。複数あれば箇条書きでも段落でもOKです。終わったら「以上」「終わり」などで合図してください。

ユーザーが「スキップ」「なし」と回答した項目は記録しない。

### 4. 既存内容との衝突を確認する

対象項目に既存内容がある場合は、書き込み前に **必ず** AskUserQuestion で確認する:

```
質問: 「やったこと」には既に内容があります。どう記録しますか？
- 追記: 既存の末尾に改行で続けて追加（推奨。やったことや補足は1日に何度も追記する想定）
- 上書き: 既存内容を消して新しい内容で置き換える
- キャンセル: この項目は今回は記録しない
```

既存が空（初回入力）の場合はこの確認はスキップしてよい。

### 5. スクリプトで書き込む

確定した内容を項目ごとに `write` サブコマンドで保存する。本文は改行を含むことがあるのでヒアドキュメントやシェルクォートに注意。Bash例:

```bash
python3 ~/.claude/skills/daily-report/scripts/write_report.py write \
  --field done \
  --mode append \
  --content "- AAAをマージ
- BBBのレビュー対応"
```

### 6. 結果を報告する

書き込み後、保存先パスと反映した項目を短くユーザーに伝える。複数項目を更新した場合はまとめて1回でよい。例:

> `~/daily_report/2026-05-11.md` に「やったこと」を追記しました。

## 日付の扱い

- デフォルトは今日（システムローカル時刻）
- 「昨日の日報」「2026-05-09の日報」のように別日を指定された場合は `--date` を渡す
- ファイル名は常に `YYYY-MM-DD.md`

## 完全フローの例

### 例1: 全項目を新規記録

ユーザー: 「日報」

1. AskUserQuestion(`multiSelect: true`, 選択肢: 今日やること / やったこと / 明日やること / その他補足) で対象を確認
2. `read --date <today>` で既存確認
3. 選択された各項目について、本文をチャットで質問 → ユーザーが回答
4. 既存ありの項目は AskUserQuestion で追記/上書き/キャンセル を確認
5. 項目ごとに `write` を実行
6. 完了報告

### 例2: 一部項目のみ

ユーザー: 「日報 明日やること」

1. 対象は `tomorrow_plan` のみと判定（AskUserQuestion不要）
2. `read --date <today>` で既存確認
3. 「明日やることを教えてください」とチャットで質問 → 回答受領
4. 既存ありなら追記/上書き/キャンセルをAskUserQuestionで確認
5. `write --field tomorrow_plan ...` を実行
6. 完了報告

### 例3: 過去日付の追記

ユーザー: 「昨日の日報 やったこと 追加で〇〇」

1. 対象日 = 昨日、対象項目 = `done`、追加本文 = 「〇〇」
2. `read --date <昨日>` で既存確認
3. 既存ありなら確認、無ければそのまま追記
4. `write --field done --date <昨日> --mode append --content "〇〇"`
5. 完了報告
